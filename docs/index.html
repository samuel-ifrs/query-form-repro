HTTP QUERY: <form> vs fetch() — live demo

Repro for whatwg/html#12594. Each form and the fetch button submit to a small QUERY-aware echo.php, which reports the HTTP method it actually received.

1. HTML forms (no JavaScript)

Three identical forms; only the method attribute differs. Click each and read the big method on the result page.

2. fetch() for comparison

Same QUERY method via fetch(). Because echo.php allows QUERY in its CORS preflight, this cross-origin call goes through (a preflight OPTIONS fires first — see DevTools → Network).

(result appears here)
Expected: the method="query" form makes the server report method GET (its body dropped) — an unknown form method falls back to GET. The fetch() call reports QUERY. That contrast is the gap #12594 is about: QUERY is reachable from script, but not from declarative forms.
